Software Architecture

Mastodon is a Ruby on Rails application, like Course Manager. It can run on standalone servers or a containerized platform like docker or kubernetes. In your lab environment, Mastodon is running as a set of docker containers on the "Mastodon" VM.

Mastodon requires a PostgreSQL server to store its data. In this environment, we are using Azure PostgreSQL Flexible Server to host our backend database. There is one database server in Azure shared between all the labs provisioned for this course.

Installation Process

This section describes the process used to set up Mastodon in this lab environment. If you were setting up a production instance of Mastodon for long-term use, the process would be different.

On the previous page, you ran a script to provision the infrastructure required for Mastodon. The provision script does the following:

  1. Verifies the course's shared database server is set up in Azure or provisions a new one if necessary
  2. Provisions this lab's Mastodon database and DB user account
  3. Creates a Sendgrid API key so that emails can be sent from the application
  4. Generates random identifiers for this instance of Mastodon
  5. Adds the Mastodon server to this environment

On this page, you ran a script to install and configure Mastodon. The installation script does the following:

  1. Configures the Mastodon server settings based on the results of the previous step
  2. Creates the tables and other database objects required for Mastodon
  3. Creates and runs the Mastodon server containers
  4. Creates the initial Mastodon user accounts

Advanced Topics: Admin CLI

This is an optional advanced topic. Click to the next page to skip it.

Most Mastodon administrative functions are performed through the web UI. However, there is also an admin CLI you can run on the Mastodon server, called tootctl

To use tootctl, you'll need to connect to the Mastodon server with SSH.

Click here to start an SSH session

The basic format of a tootctl command is tootctl COMMAND, where COMMAND is one or more words that represent the action you are taking.

In our environment, Mastodon is running in docker, so we'll need to prefix our commands with docker exec, the name of the container, and bundle exec, which loads the runtime dependencies.

Disabling Registrations

One use case for tootctl is to prevent new users from self-registering at the site

To disable registrations, type the following command into the SSH terminal:

docker exec web bundle exec tootctl settings registrations close

Permanently Deleting a User

Mastodon's web UI allows you to freeze or suspend a user, but it does not permit you to permanently delete a user from the database.

To delete a user's account entirely, type the following command into the SSH terminal:

docker exec web bundle exec tootctl account delete USERNAME

Replace the word USERNAME with the user's actual username.

To test this out, you can delete the troll user we froze in a previous session, whose username is %{metadata:troll_username}

Advanced Topics: Translation Bot

This is an optional advanced topic. Click to the next page to skip it.

Mastodon provides both REST and streaming APIs that can be used to develop a variety of bots or integrations.

In this exercise, you will use these APIs, along with a translation API from Azure, to run a bot that will translate Spanish posts to English and vice versa.

Exercise Setup

First we'll need to set up a user account for our bot and an API token to use with Azure's translation services. Click the button to set things up.

Setup Translator

Now we'll switch to our dev VM. Click the button to switch to the VM and launch Visual Studio Code.

Launch Visual Studio Code

In Visual Studio Code, click Terminal, then New Terminal. In the new terminal window, type git pull to pull the latest code from the course-manager-examples repository.

Configure Translator

In the left panel of the code editor, you will see the directory structure.

Locate the directory scripts/mastodon/translation_bot. In this directory, you will see a file called bot.rb, which contains the ruby code that will run our bot.

The code depends on environment variables to work properly. To configure the environment variables, do the following:

  1. Open the file called dotenv.example
  2. Click File, then Save as, and save with filename .env
  3. In the file, paste %{metadata:lab_fqdn} into the line starting with MAST_FQDN
  4. Follow the steps below to get a Mastodon token to use with the bot

To connect to Mastodon, you'll need to log in as the translator user and generate a token.

Launch Firefox

Now enter the credentials for the translator account

Click the "Sign in" button. Copy and paste the username and password from the table.

Username translator@bot.fake
Password %{metadata:translator_password}

After you have signed in, click Preferences, then Development, then New application

Enter anything you like for application name, leave the other settings unchanged, and click Submit

Copy the token listed under Your access token and paste into the .env file in the line starting with MAST_TOKEN

Running the bot

Now that we have configured the bot, it's time to run it.

In your vscode terminal, type the following commands:

cd scripts/mastodon/translation_bot
bundle exec ruby bot.rb

If we configured everything correctly, the bot will connect to Mastodon and begin to listen for toots.
